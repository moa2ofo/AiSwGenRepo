Detailed Description

Current monitoring derating API and support types.

This module defines the types, macros, calibration tables and APIs used to compute current derating limits for DC-link and line currents based on supply voltage and ECU temperature. The main entry point is CurrentMon_Derating, which reads the measurement interfaces, evaluates the derating curves and exposes the resulting limits via global output variables.

Utility macro returning the minimum of two values.

This macro is used to combine different derating contributions by selecting the most conservative (lowest) limit. In this module it is primarily used by CurrentMon_Derating to compute the final DC current limit CurrentMon_IdcLim_s16 as:

CurrentMon_IdcLim_s16 = CURRENTMON_MIN(limit_IdcVdc, limit_Idctemp);

◆ CurrentMon_mathApplPoint_t

typedef struct CurrentMon_mathApplPoint_s CurrentMon_mathApplPoint_t

2D point for derating curve (x, y).

This structure describes a single point of a 1-D derating curve used by CurrentMon_SetDeratingLimits_s16. The field x_s16 represents the input variable (e.g. voltage or temperature), while y_s16 represents the corresponding limit (e.g. current limit).

Typical curves:

  • Vbat vs. DC current limit (CurrentMon_IdcVdcTab)
  • ECU temperature vs. DC current limit (CurrentMon_IdcTecuTab)
  • ECU temperature vs. line current limit (CurrentMon_IlineTecuTab)

◆ CurrentMon_SetDeratingLimits_s16()

sint16 CurrentMon_SetDeratingLimits_s16 ( const sint16 x_var,
const CurrentMon_mathApplPoint_t * tab,
const uint8 n_points )

Evaluate a 1-D derating curve and return the corresponding limit.

This helper function interpolates a derating characteristic described by an array of CurrentMon_mathApplPoint_t and returns the associated limit for the given input value x_var.

Typical usage (inside CurrentMon_Derating):

  • Voltage-based DC current derating using CurrentMon_IdcVdcTab to compute limit_IdcVdc.
  • Temperature-based DC current derating using CurrentMon_IdcTecuTab to compute limit_Idctemp.
  • Temperature-based line current derating using CurrentMon_IlineTecuTab to compute CurrentMon_IlineLim_s16.

From the formal specification (ACSL) this function is modeled as pure: it does not modify any global state and always returns a non-negative result within an implementation-defined upper bound.

Parameters
[in]x_varInput variable for which the derating limit is required (e.g. supply voltage in mV or ECU temperature in °C).
[in]tabPointer to the derating curve (array of points).
[in]n_pointsNumber of valid points in tab.
Returns
Derated limit associated to x_var (non-negative).
Precondition
tab must not be NULL.
n_points must be greater than zero.

sint16 TempMeas_get_filtEcuTemp_s16 ( void )

Read the filtered ECU temperature.

This function returns the ECU temperature in degrees Celsius, already filtered and scaled as required by the derating logic.

In the formal model (Frama-C) it is treated as a pure function: it does not modify any global state and simply returns the current value of the ghost variable CurrMon_Tecu_degC.

Returns
ECU temperature in degrees Celsius [°C].
Note
The returned value is used by CurrentMon_Derating as input for both temperature-based derating curves:
  • CurrentMon_IdcTecu_tab (DC current limit vs. TECU)
  • CurrentMon_IlineTecu_tab (line current limit vs. TECU)
